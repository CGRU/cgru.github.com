Autosave manager.

Autosave manager window configures the automatic saving of your scene:

 

Enable autosave - turns it on or off.
help - open this HTML page.
Autosave every "#n" nimutes - sets the autosave time.
Selections - set autosave to count selection mode.
Show confirm dialog - raises a dialog when the time is about to save, so you will be able to choose whether to save or not.
Add to recent files list - adds an autosave name to maya " File -> Recent Files" menu.
Files maximum - sets the cycle quantity of autosave files.
Name padding - sets the quantity digits in the number or the filename.
Save in another directory - sets the directory to place autosave files. If this flag is off autosave manager will place files in the directory you work in.
browse - browse path to the autosave directory.

history backup section:
enable backup - turns backup on or off.
backup every "#n" saves - set the frequency of backup.
backup directory - set the directory to place backup files.
browse - to browse path to the backup directory.

By default the autosave is off. You can use autosave to save you scene periodically. Choose the frequency of autosaving by setting up the autosave time. Maya will save your scene every "n" minutes with the adjusted name. Or you can use autosave in "Selection" mode. In this mode it counts every maya selection changes.

It adds special autosave symbol - "@" and a save number with the specified padding. For example - "test.@0006.mb". And prints full autosaved path - "autosave result - c:/temp/autoSaveDir/temp.@0006.mb". When save number is equal to "files maximum" manager sets it to zero. So autosave is a cycle procedure, whith overwrites old autosave files.

Every time user changes selection manager compares the time of last autosave and the current time. So if you do not doing anything (if you went away, for example) manager will not work! It is very useful, because if you was away for a long time manager will not overwrite you last saves with the same scene.

History Backup - spacially designed for backup you saves history. Unlike autosave it never overwrite files. Backup adds to autosave name current date and time and stores it in a specific backup directory. So there is no way for name coincidence. It is the reason for the permanent enlarging of backup directory. You should clean it up periodically ( about every several mounths or once a year at least). The advantage of using history backup is that you can always do the steps back to the history crearing you masterpieces.
