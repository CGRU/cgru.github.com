RULES

RULES is a WEB engine to view/modify folders structure, designed for CG projects tracking.
  • Comment scenes, shots, folders.
  • Edit shots (folders) statuses, progresses, artists and tags.
  • Subscribe channels (folders), receive news on changes.
  • View images sequences, draw annotations.
  • Create, view and comment dailies.
  • Setup users, groups, permissions.

It is beta version till CGRU 2.0.

Online Version

Rules online: http://cgru.info/rules

Online version has some limitations:

  • Your machine does not have an access to its filesystem, so browser can`t open folders and files.
  • There is no Afanasy, so it can't send jobs, you can't create dailies from images sequences.
  • There is no video and image conversion tools, you can't generate thumbnails.
  • Web server does not have authentication mechanisms, so you can access files typing direct links to them. If you do not have permissions to some location, it will be just hidden in RULES GUI. But such public place is not designed to keep any private information.

Header

Settings

You can setup colors, view options and login here. All setting will be stored in your browser.

Assets Links

All founded assets are shown here with links.

News Counter

Number of new displayed here. News are updated every minute. You can click counter for update and open news side panel.

Navigation

Options

You can setup what to display in folders items. Only first wold on annotation will be shown.

Folders Tree

This is your filesystem tree.

Content

Thumbnail

This is location thumbnail.

Status

Status consists of:

  • Annotation, it first world can be shown in folders tree.
  • Progress percentage. You can set it to '-1' to exclude from average calculation.
  • Finish time.
  • Artists. Any registered users can be assigned to location.
  • Tags. Any location can have its own tags setup.
  • Color, just to decorate location in GUI.

Asset

It some custom viewer. You can setup rules to find assets automatically or point asset directly. Asset can have own html and javascript code, do and show anything.

Files

Location files and folders. If there is some asset founded, files are not shown, if asset does not have special option.

Body

Body is a some editable page. Can be used for anything. For example project can have technical info, FPS, input and output format settings. Scene and shot can have tasks description.

Comments

This is location comments. If you want to upload something within comment, you can drag and drop files in upload field, but not start to upload them. Such files will be uploaded with command and attached to it:.

Side Panel

News

News are shown here, with links. To delete link, double-click '-' button. To delete all news, double-click 'Clear News' button.

Click 'Channels' to list subscribed locations links. To delete channel, double-click '-' button.

Playlist

It is just global bookmarks. User can bookmark any location (shot) in browser in common way. Other users (browsers) will not see that bookmarks. To make a global bookmark add a link to playlist.

For example we have a viewing room with a computer with a huge display. To show shot (images sequence) user puts it in a playlist. Than, on viewing time, user can quickly open his sequence. Before RULES, each user was to find (browse) the sequence to view.

Upload

This is a way to upload files. Any location (shot, asset) can have its own relative upload path. By default shots are configured to put files in 'UPLOAD/date_user' relative folder.

For example, to put file in 'SHOT/WORK/ART' folder and comment it in a shot, you should go to that folder, drag and drop file(s) in upload field, go back (upper) to the shot, and make a comment. File will be uploaded in 'SHOT/WORK/ART' as it was dropped there, link and a thumbnail to the file will be attached to comment.

Permissions

This is where administrators can control users permissions. Permissions are stored in the same way as Apache server understands. To if you dined some folder to some users (groups), direct links will not works too. No way to hack RULES unless you can hack Apache.

Both, and Apache and RULES GUI understands permission setup when you can deny upper folder but give access to some child folder(s). So you can give access only to special shots of a projects.

Footer

Info

This is status line. It shows current path after navigation. RULES understands CGRU Pathsmap and can transfer paths from server to local client. You can open that path.

Version

On startup server reads cgru/version.txt file, like any other CGRU tool. You can write anything there to display some custom information.

Log

System logs are stored there. Needed to debug. You can see time with milliseconds for each logged action. So you can monitor what needs more time and slows down entire system.

Player

JPEG images sequence player. You can paint annotations.
