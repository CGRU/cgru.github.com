Afanasy User

User is created on a new job from user that does not exist. Or you can create user manually unsing CLI:

afcmd uadd username

User Attributes:

Name
name
Each user should have an unique name. If not some reason artists has the same login name, you can user AF_USERNAME environment variable to override it.
Need
(n/a)
Not solved user with greatest need value will get next free host (if he or his jobs are able to do it). Users are simply sorted by this value in solving function. Need depends on user's priority and number of running hosts.
Host Name
host_name
Host name where the last job was send from.
Jobs Number
jobs_num
Number of jobs the user has.
Running Jobs Number
running_jobs_num
Number of currently running jobs the user has.
Running Tasks Number
running_tasks_num
Number of currently running tasks the user has.
Registration Time
time_register
Time when the user was registered.
Activity Time
time_activity
The last user activity time.

User Editable Parameters:

Priority
priority
User with greater can have more running tasks number (get more render hosts).
need = pow( 1.1, priority) / (runningtasksnumber + 1.0)
Each priority point gives 10% hosts bonus (running tasks number).
Maximum Running Tasks
max_running_tasks
Maximum number of running tasks user can have.
Hosts Mask
hosts_mask
User can run only on renders which host name matches this mask.
Exclude Hosts Mask
hosts_mask_exclude
User can not run on renders which host name matches this mask.
Error Retries
errors_retries
Default Error Retries value for user jobs.
Errors Avoid Host
errors_avoid_host
Default Errors Avoid Host value for user jobs.
Errors Task Same Host
errors_task_same_host
Default Errors Task Same Host value for user jobs.
Errors Forgive Time
errors_forgive_time
Default Errors Forgive Time value for user jobs.
Jobs Life Time
jobs_life_time
Default Life Time value for user jobs.
Annotation
annotation
Annotate user GUI item.
