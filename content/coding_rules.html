Coding Rules:

  1. Identation - TAB, default 4 spaces length is normal (but you can vary).
  2. Code alignment - SPACES.
  3. Variable names: variable_name.
  4. Function names: functionName.
  5. Class names: ClassName.
  6. Use prefixes to make code more readable:
    • "i_" - Input variables.
    • "o_" - Output variables.
    • "m_" - Class members.
    • "ms_" - Static class members.
    • "g_" - External variables.
    • "v_" - Virtual functions.
  7. Do not use "!" as NOT, it is not noticeable while code passing view, much more easy to notice "false == "
  8. Use "const &" to pass complex types as function parameters to not to copy class instance.

Example:


// Function with long parameters list:
bool someFunction(
		std::string & o_status,
		const std::string & i_param1,
		const std::string & i_param2,
		const std::string & i_param3,
		const std::string & i_param4
	)
{
	...
	if( false == variable_name )
	{
		o_status = "error";
		return false;
	}
	return true;
}
